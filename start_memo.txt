# ★ 改訂版：メイリンゲーム開発サイクル

## STEP 1. 開発

Godot / JSON / GDScript / docs の作業をする。

---

## STEP 2. Git に保存して GitHub と同期

```bash
cd /c/GodotProjects/meirin-battle-godot

git add .
git commit -m "update: <具体的な内容>" 
git push
```

---

## STEP 3. ChatGPT 用 “URL 一括リスト” を生成

```bash
git ls-files '*.gd' '*.json' \
  | sed 's|^|https://raw.githubusercontent.com/HirokiTakah/meirin-godot-game/main/|' \
  > urls_raw.txt
```

---

## STEP 4. ChatGPT に読み込ませる

web ツールを使用して、下記の GitHub リポジトリを読み込んでください。
HTML ではなく、必ず raw.githubusercontent.com の生データを取得してください。

https://github.com/HirokiTakah/meirin-godot-game

まず最初に、以下の設計JSONを必ず読み込んでください（仕様書の正として扱う）。
1) docs/design_upstream_core.json
2) docs/design_midstream_story.json
3) docs/design_midstream_system.json
4) docs/meirin_oop_bundle.json
5) data/meirin_oop_game_ref.json

次に、開発工程表（正規ロードマップ）として
docs/dev_plan_pipeline.json
も raw で読み込んでください。

その後、わたしが手動で貼り付ける「raw リンク一覧（urls_raw 手動版）」に含まれる
すべての JSON / GDScript を順番に読み込んでください。
（自動で urls_raw.txt を取りに行かないこと。必ずこちらが提示するリンクだけ読み込むこと）

読み込み後は、次のルールに従います。

・上記 6 つの JSON（設計5種＋工程表）を、実装より優先して参照すること  
・旧じゃんけんロジック（GDevelop / 旧Godotの三すくみ判定）は使用しないこと  
・バトルは RPGターン制（HP / 心ゲージ / フォームゲージ）で実装すること  
・設計JSONとコードが矛盾する場合は、設計に合わせて修正案を提示すること  
・docs/dev_plan_pipeline.json の工程に沿って開発を進めること  
（作業フェーズとタスクIDを明確にした上で回答する）

準備が整ったら、
「設計JSON5種・工程表JSONを読み込みました。次に raw リンクを提示してください。」
とだけ返答してください。
