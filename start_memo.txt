# STEP 1. 開発（ローカル作業）

Godot / JSON / GDScript / docs の作業を進める。

---

# STEP 2. Git に保存して GitHub と同期

```bash

cd /c/GodotProjects/meirin-battle-godot

git add .
git commit -m "update: P2_T1"
git push
```


---

# STEP 3. ChatGPT 用 URL リストを生成（3分類）

```bash

cd /c/GodotProjects/meirin-battle-godot

git ls-files 'docs/*.json' \
  | sed 's|^|https://raw.githubusercontent.com/HirokiTakah/meirin-godot-game/main/|' \
  > urls_docs_json.txt


git ls-files 'data/*.json' \
  | sed 's|^|https://raw.githubusercontent.com/HirokiTakah/meirin-godot-game/main/|' \
  > urls_data_json.txt

git ls-files 'src/*.gd' \
  | sed 's|^|https://raw.githubusercontent.com/HirokiTakah/meirin-godot-game/main/|' \
  > urls_gd.txt

```

* urls_docs_json.txt … 設計JSON
* urls_data_json.txt … ゲームデータJSON
* urls_gd.txt … GDScript（必要時）

---

# STEP 4. ChatGPT に読み込ませる（初期化：docs → data）

新しいチャット開始時に、まず ChatGPT に **「工程表に沿って開発を進めること」** を理解させる。

### （チャット開始テンプレ）





```

web ツールを使用して、下記の GitHub リポジトリを読み込んでください。
HTML ではなく、必ず raw.githubusercontent.com の生データを取得してください。

https://github.com/HirokiTakah/meirin-godot-game

このプロジェクトの作業は、必ず docs/dev_plan_pipeline.json（工程表）に沿って進めます。
以降、わたしは工程表の進捗に応じてタスクを指定します。
あなたは、そのタスクを進めるために必要な .gd を判断し、必要なファイル名を求めてください。

このメッセージの下に、docs/*.json（設計ファイル）の raw URL 一覧を貼ります。
それらをすべて読み込み、設計JSON（仕様の正）として扱ってください。

読み込み後のルールは次の通りです。

・docs 配下の JSON（設計5種＋工程表）を実装より優先して参照すること
・設計JSONとコードが矛盾する場合は、設計を優先した修正案を提示すること
・開発は docs/dev_plan_pipeline.json のフェーズとタスクに沿って進めること
・作業が進んだら、工程表にもとづいて次タスクを指示すること

上記の docs/*.json をすべて読み込んだら、
「docs の設計JSONを読み込みました。次に data の json raw リンクを提示してください。」
とだけ返答してください。


→ **このテンプレの下に、urls_docs_json.txt の内容を貼る**

```


ChatGPT が docs JSON を読み込んだら

「docs の設計JSONを読み込みました。次に data の JSON を提示してください。」

と返す。

---

# STEP 5. data JSON を読み込ませる（初期化：第2段階）

ChatGPT が要求したら、urls_data_json.txt の中身をそのまま貼るだけ。

読み込み完了後 ChatGPT は：

「data JSON を読み込みました。
工程表に沿って次の作業を指示してください。」

と返答する。

これで **プロジェクトの初期化が完了**。

---

# STEP 6. 工程表に沿ってタスクを指示（.gd は必要時のみ）

初期化が終わったら、あなたがやることは **これだけ**：

### ★（シンプル依頼テンプレ）

```
工程表の「○○」まで進みました。
次は「○○」の作業をします。

この作業に必要な .gd があればファイル名を教えてください。
必要に応じて raw URL を貼ります。
```

ChatGPT は：

* 工程表を参照し
* 次タスクに必要な `.gd` を判断し
* raw URL を要求する

あなたは urls_gd.txt を見て必要な URL を貼るだけ。

---

# ● 全体の流れ（完成版）

```
STEP1  開発（ローカルで Godot / JSON 編集）
STEP2  GitHub へ push
STEP3  URLリスト生成（docs/data/gd の3種）
STEP4  新チャット：docs → data 読み込み（初期化）
STEP5  初期化完了
STEP6  工程表に沿ってタスク指示
        → ChatGPT が必要な .gd を要求
        → raw URL を貼る
        → 修正/実装案を受け取る
```

とてもシンプルで再現性が高い形になりました。
